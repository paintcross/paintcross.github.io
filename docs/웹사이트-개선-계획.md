## 0. 한 줄 요약

**이 페이지는 "무료 앱 소개 페이지"로는 완성도가 높지만, "$9.99 Pro를 파는 상거래 페이지"로는 결제 진입점·가격 섹션·SEO·정책 페이지가 전부 없다.** 설계안의 Day 1 병목(랜딩에 Pro 없음)이 코드 감사로 그대로 확인됐고, 오히려 페이지가 4곳에서 "Free"를 강조해 Pro 도입 시 메시지 충돌까지 예약된 상태다.

## 1. 현재 상태 진단 (감사 결과 요약)

### 구조
`nav → hero → 앱 프리뷰 캐러셀 → Examples(3) → Features(6)+통계바 → Download(OS 감지·설치 가이드) → 미니 FAQ(3) → footer`

### 확인된 강점
- OS/아키텍처 자동 감지 + 릴리스 API 연동 다운로드 흐름 완성도 높음
- 10개 언어 i18n, 시스템 폰트만 사용(웹폰트 0), 외부 렌더 블로킹 리소스 0
- "no Electron, no bloat" 메시징이 hero·프리뷰 캡션·download sub에서 일관 반복
- `checkout/success/` 결제 성공 페이지(라이선스 키 표시·복사·활성화 4단계 안내) 이미 존재

## 2. 개선 계획 (우선순위별)

### P0 — 첫 매출 차단 해제 (설계안 Day 1~2와 직결)

**P0-1. Pricing/Pro 섹션 신설** — Features와 Download 사이 권장
- 무료 vs Pro 비교표: **앱의 실제 Pro 게이트(17개 기능) 기준으로 작성** (설계안의 "내보내기 3종 + 고급 필터"는 실제보다 좁은 서술 — 근거: Paint.Cross `src/lib/premium.ts:4-28`)
  - 내보내기: WebP·PDF·OpenRaster + 커스텀 DPI
  - 편집: 고급 조정 필터(레벨·컬러 밸런스), 블렌드 모드 16종, 클립 마스크, 시메트리
  - 브러시·레이어: 프리미엄 브러시 3종(Charcoal·Spatter·Scatter), 레이어 그룹, 무제한 레이어, 고해상도 캔버스
  - 컬러·보조: 컬러 하모니 적용, 팔레트 추출, 커스텀 그리드·가이드·눈금자, 마커 확장 범위(1–9999)
  - 공통: 라이선스(v1.x 평생) + 개발자 직접 지원
  - 표에는 대표 5~6개 + "and more"로 압축하고, 전체 목록은 툴팁/펼침 또는 FAQ로
- 가격 표기: **"$9.99 — one-time, launch price"** ("launch price" 배지 권장. ⚠️ 인상 후 가격이 설계안은 $14.99, 앱 저장소 `docs/premium-feature-strategy.md`는 v2 $19.99로 불일치 — 미래 가격을 암시하는 문구·JSON-LD를 넣기 전에 한쪽으로 통일할 것)
- ⚠️ **"평생" 문구는 v1.x 한정으로 정확히**: 라이선스는 `lifetime_1x`(major 1 한정)로 앱이 강제하며(`src-tauri/src/license.rs:190-202`) v2는 재구매 대상. 앱 내 문구도 "Lifetime ownership of version 1.x". 랜딩에서 무조건 "forever"/"평생"으로 쓰면 환불 분쟁 소지 → **"Lifetime license for v1.x"** 계열로 통일
- CTA: "Get Pro" → Dodo Payments 결제 링크(앱 내 결제와 동일 흐름) 또는 "다운로드 후 앱에서 업그레이드" 안내 — **Dodo 정적 결제 링크가 있으면 웹에서 바로 결제 → `checkout/success/`로 리다이렉트되는 기존 자산을 드디어 활용하게 됨**
- 설계안의 랜딩 한 줄 반영: *"Native on Mac, Windows, and Linux. One price — $9.99. No subscription, no account, no bloat."* (원안의 "One price, forever"에서 "forever"는 위 v1.x 이슈로 제외하거나 "v1.x lifetime"으로 대체)

**P0-2. "Free" 메시지 충돌 해소**
- hero CTA `Download — It's Free`(725) → `Download Free` 유지하되 신뢰 배지를 `✓ Free core · ✓ No account · ✓ No subscription`으로 재구성(729–732, 주석 처리된 항목 731 활용)
- 통계바의 어색한 `Free / Free` 이중 표기(842–843) → `$9.99 Pro — one-time` 또는 `No subscription`으로 교체
- "Pixelmator went subscription. We didn't." 각도는 Pro 섹션 서브카피 후보

**P0-3. FAQ에 가격·라이선스 항목 추가**
- "Is it really free?" (무료 코어 + Pro 게이트 설명), "Is Pro a subscription?" (일회성·평생·기기 정책), "How do I get my license key?" (checkout/success 흐름 연결)

**P0-4. 결제 성공 페이지 정비** (`checkout/success/index.html`)

### P0-A — 앱 저장소(Paint.Cross) 연계 작업

> 이 문서는 웹사이트 계획이지만, 랜딩에 가격을 붙여도 앱이 안 받쳐주면 반쪽이 된다. 아래 4건은 별도 저장소(`Tools/Paint.Cross`) 작업으로, P0과 같은 시점(Day 1~2)에 처리.

**P0-A1. UpgradeModal에 가격 표기** (Day 1 — 랜딩 P0-1과 같은 날)
- 현재 앱 UI 어디에도 $9.99가 없음 — 버튼은 "Get Pro License"뿐이고, 사용자는 Dodo 결제 페이지에서 처음 가격을 봄
- `src/components/UpgradeModal.tsx`에 **"$9.99 · Launch price · One-time"** 표기 추가 + `upgrade.price.*` 키를 10개 로케일(`src/i18n/locales/*.ts`) 전부에 추가 (`upgrade.*` 키는 현재 10개 언어 × 125개로 완비된 구조라 같은 패턴으로)
- 가격 하드코딩 대신 빌드타임 env(예: `VITE_PRO_PRICE`) 분리 권장 — 추후 인상 시 재번역 없이 교체

**P0-A2. 체크아웃 URL live 전환** (Day 2 — 실결제 리허설의 전제)
- `.env.local`의 `VITE_CHECKOUT_URL`이 현재 **Dodo test 호스트**(`test.checkout.dodopayments.com/buy/pdt_...`) — live 결제 링크 확보 후 릴리스 빌드에 주입, `npm run release:check` 가드 통과 확인
- P0-1의 웹 직접 결제 링크도 이 live URL을 그대로 사용
- 리허설 시 확인: Dodo live가 `checkout/success/`로 리다이렉트할 때 `license_key` 쿼리 파라미터를 실제로 붙이는지 + 수수료 차감 후 건당 실수령액(설계안 검증 항목)

**P0-A3. 앱 → 웹사이트 링크 추가**
- 현재 앱의 유일한 외부 링크가 체크아웃 URL 하나 — Help 메뉴(`src/components/MenuBar.tsx:517-524`)에 "Website" 항목 추가해 랜딩 페이지로 연결

**P0-A4. 지원 이메일 주소 확정 (웹+앱 동시)**
- P0-4의 checkout/success 3곳 외에, 앱 쪽 `src/lib/licenseRecovery.ts:1`에도 같은 개인 Gmail이 하드코딩(자동 재발송 백엔드 없음, 수동 복구 전용)
- 앱은 **재빌드가 필요**하므로 주소 결정을 Day 1~2로 앞당길 것 — 웹은 언제든 고치지만 배포된 바이너리는 못 고침

### P2 — 구조 수선 (i18n·API·모바일)

**P2-1. i18n 리팩터링** — Pricing 섹션 추가 *전에* 하면 이후 비용이 급감
- 현행: 거대 사전 + 패치 객체 5개 `Object.assign` 누적 + `applyLanguage()`에 ~70개 ID 수동 `setText()` — 3곳 동기화 필요, 누락 시 조용히 영어 폴백
- 개선: `data-i18n="key"` 속성 기반 단일 루프로 전환, 사전은 언어별 단일 객체로 병합
- 버그 수정: 캐러셀 `data-title`/`data-caption` 하드코딩 제거 → `data-title-key` 경로 살리기(1842–1853의 죽은 키 부활)
- 갭 보충: zh·es·ar·hi·ja·fr·de·pt의 Examples 섹션 키 8언어 × 8키
- 아랍어: `applyLanguage()`에서 `documentElement.dir = lang === 'ar' ? 'rtl' : 'ltr'` + RTL 레이아웃 점검

**P2-2. 모바일 수선**
- 모바일 내비: 햄버거 또는 최소한 Download 링크 노출 (현재 앵커 내비 전멸)
- Android UA를 Linux로 오인(1201) → 모바일 감지 시 "데스크톱 앱입니다 — 링크를 PC로 보내세요" 안내(이메일/QR)로 전환. 모바일 방문자는 런칭 트래픽의 상당수가 될 것(Reddit 모바일)
- 캐러셀 모바일 렌더링 점검(스크린샷상 슬라이드 겹침 의심)

**P2-3. GitHub API 호출 다이어트**
- 다운로드 카운트: 페이지당 최대 11회 → `localStorage` 캐시(TTL 1h) + `releases/latest` 1회로 축소, 전체 합산은 백그라운드 1회
- `cache: 'no-store'` 제거, 403 시 마지막 캐시 값 표시

### P3 — 성능·품질 마감

- **이미지**: 참조 PNG 7장 WebP 전환(645KB → 예상 ~200KB), 캐러셀 2~4번·Examples 3장에 `loading="lazy"` + `width/height` 속성
- **저장소 청소**: orphan 삭제 — `app-screenshot.png`, `hero-layers.png`, `hero-markers.png`(~342KB), `test.js`. `docs/superpowers/`·`.abacusai/`·`docs/*.md` 공개 여부 결정(비공개면 별도 저장소나 .gitignore)
- **접근성**: `prefers-reduced-motion` 시 캐러셀 자동재생 정지 + 애니메이션 축소, 캐러셀에 일시정지 버튼, 언어 메뉴 화살표/Escape 키보드 패턴, `#linux-guide-trigger`를 `<button>`으로, 비활성 슬라이드 `aria-hidden`, `<main>` 랜드마크

## 4. 측정 (첫 2주 검증과 연결)

설계안의 "검증 필요" 항목을 페이지에서 측정하려면 최소한의 분석 도구가 필요:
- 프라이버시 친화 분석(예: GoatCounter·Plausible 셀프/무료) 1개 도입 검토 — "No account, no tracking" 포지셔닝과 충돌하지 않게 쿠키리스 선택, privacy 페이지에 명시

- 런칭 채널별 UTM(`?ref=reddit` 등)은 링크 생성 시 규칙만 정해두면 됨
